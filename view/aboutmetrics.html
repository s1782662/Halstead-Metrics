HALSTEAD METRICS

          Halstead complexity measures are software metrics introduced by Maurice Howard Halstead. In 1977, he developed it as
a part of his treatise on establishing an empirical science of software development. Halstead made the observation that metrics of the software should reflect the implementation or expression of algorithms in different languages, but be independent of their execution on a specific platform. These metrics are therefore computed statically from the code.

Let us consider the following C program designed to calculate factorial:

#include <stdio.h>
int main()
{
 int c, n, fact = 1;
  printf("Enter a number to calculate it's factorial\n");
  scanf("%d", &n);
  for (c = 1; c <= n; c++)
    fact = fact * c;
  printf("Factorial of %d = %d\n", n, fact);
  return 0;
}

After running the code through our app, η1 = 20, η2 = 21, N1=53, N2=37

LOC

          Source lines of code (SLOC), also known as lines of code (LOC), is a software metric used to measure the size of a computer program by counting the number of lines in the text of the program's source code. SLOC is typically used to predict the amount of effort that will be required to develop a program, as well as to estimate programming productivity or maintainability once the software is produced.
